    • Go使用总结

    代码组织

    go的项目通常对应一个模块，模块下有很多package，package对应一个文件夹，需要在项目的根目录下创建go.mod来指定module的路径，该路径不但是包引用的prefix path，同时也告诉go如何去获取网上资源

    go install会将程序打包成二进制，存放目录是$GOPATH/bin, $GOPATH默认是$HOME/go

    go build可以构建package，构建结果放入cache中，所以不会有文件生成

    go mod tidy将新的项目依赖写入go.mod文件,同时删除不需要的依赖

    go的依赖统一安装，如果多个项目依赖相同版本的module，那么该module不会被重新安装，这个比单独项目来维护依赖节约很多空间
    甚至为了节省空间，可以把安装的依赖完全删除
    go clean -modcache

    基础测试

    go测试，一般在你的package下创建一个结尾为_test的文件，引入testing包，创建函数，格式为func TestXXX(t *testing.T){},在函数内部调用t的Fail或Error方法来表示错误，最后在package下运行go test

    语法要点（基于GOTOUR）

    基础

    go不支持++i语法 i++是语句不是表达式，所以j=i++是错误的

    go的返回值可以命名，return语句不带返回值默认返回命名返回值

    :=语法只能在函数体内被使用

    go的基本变量：

    var也可以同import一样使用（）来给一组变量赋值

    go在赋值时需要显示转换

    go除了var还有constant

    switch也可写一条赋值语句，switch后面什么都不写可以作为if else语句使用

    defer关键字使得后面的函数调用在函数返回时才执行，但是函数的参数会先evaluate，通常可以进行一些读写关闭的操作

    多个defer语句执行顺序是LIFO

    定义struct

    构建struct
    Name {value...}或
    Name {key:value}

    自我备注：对于不包含field的struct，我们通常不会去创建一个struct，比如EOF错误，我们一般不需要去初始化

    go中可以通过.的方式直接通过指针来访问field，无需dereferencing

    数组创建的格式是[n]T

    slices指向的是数组，所以对slice的变动会影响到底下的数组

    slice literal通过[]T{value…}来创建，它首先创建数组，然后再创建一个slice

    通过len得到slice的长度，通过cap得到底下的数组的长度，起始位置是从slice起始位置计算的

    slice的zero value是nil

    通过make创建slice，实现动态大小的数组

    make([]T, len) 创建长度为len的slice

    make([]T, len, cap)创建长度为len，而底下的数组大小为cap

    通过append(s []T, vs …T) []T 来添加元素

    for i, v:= range slice来遍历slice

    for i:= range slice 只获取index

    创建map的方法 m := make(map[keyType]ValueType)

    或literal方式 var m = map[keyType]ValueType { key: val, }

    如果valueType不是多重嵌套的结构，可以只写括号，无需ValueType{}，比如常见的struct

    判断key存在与否：ele, ok = m[key]

    删除delete(m , key)

    方法比函数多一个receiver argument：

    要修改struct值，需要传递pointer receiver
    记住所有argument包括receiver argument都是传值的

    对于含有pointer receiver的方法，在调用时无需(&v).method，只需要v.method,go会自动将其转换成pointer，但是对于普通函数argument必须一致

    同理，如果receiver argument定义的是value，那么当你使用pointer.method时，go会自动转换成value。总之，receiver argument使用起来是比较随意的，但是普通的argument必须保持完全一致

    选择pointer receiver，好处：

    1. 可以修改参数值

    2. 避免copy过程

    定义interface:

    接口

    interface可以看作（value，type）的tuple

    go语言中对于interface的implementation是隐式的，没有类似implement的关键字，当一个struct“恰好”拥有了一系列interface定义的方法，那它就是implement了这个interface了，所以具体的实现可以同interface脱离开来，放在不同的package下

    对于一个interface type的var，如果其value是nil值，也就是未被初始化的值，调用其方法并不会抛异常，receiver只会得到nil值，通常通过程序本身来处理nil值。

    但是如果是为nil的interface，调用其方法会报错，因为此时interface的type为nil，所以无法找到对应的方法,同时value当然也为nil。

    empty interface，书写为interface{}，通常在参数类型不确定的时候使用，如fmt.Printf函数

    type assertion,用来获取interface底下具体的值，注意这跟类型转换不一样，如果是int，那就得用int，不能用float64不然会报错
    t := i.(T)
    错误的assertion会panic
    更安全的方式
    t, ok := i.(T)

    type switch，注意type是关键词，类似于进行多次的type assertion

    内置interface：

    error，有一个Error方法

    fmt.Stringer interface，有一个String方法，fmt的package在打印时会调用这些interface的方法

    t Type这个代表t是pointer类，注意同p还可以用来dereference

    并发

    chanel的创建：
    c := make(chan T)

    从channel发送和接受值
    v <- ch
    ch <- v

    创建buffered channel:
    c := make(chan T, buffer_size)

    buffered channel会阻塞，分以下情况：

    1. 当buffer满了还要加入新的元素时
    2. 当buffer是空的但是对方还在等待接受时

    close(channel)来通知receiver管道已经关闭，没有值需要传输

    v, ok := <-c
    可以来判断管道是否关闭，ok为false为关闭

    for v:= range c { }
    自动检查管道的关闭

    注意，receiver不应该close，否则发送时会panic,只有sender才可以close

    select语句会选择非阻塞的channel，如果都是非阻塞的，则会随机选择，可以通过select的阻塞特性来完成程序的逻辑

    注意：select每次只能跑一个

    go提供了sync.Mutex类，它具有Lock和Unlock方法，可以确保代码的互斥Mutual Exclusion
